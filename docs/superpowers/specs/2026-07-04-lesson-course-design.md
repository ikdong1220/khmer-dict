# 캄보디아어 사전 v3.0 — 상황별 레슨 코스 재설계

날짜: 2026-07-04
대상 파일: `캄보디아어사전_v2.0.html` (v3.0으로 업그레이드), 빌드 스크립트 `scripts/build_dictionary.mjs`

## 배경

기존 v2.0(플래시카드+SRS+4지선다 퀴즈+회화 탭)은 요청자가 의도한 방향과 다르다는
피드백을 받았다. 요청자는 무작위 암기 카드가 아니라, 상황별 표현을 순서대로
익히는 레슨 코스 형태를 원한다. 또한 이 앱은 요청자 개인 전용이 아니라 다른
사람도 함께 쓰는 범용 학습 도구로 취급한다.

데이터 조사 결과 20,000단어 중 96.8%(19,360개)는 OPUS KDE4/MediaWiki 코퍼스에서
가져온 소프트웨어 UI 문자열이며 실생활 대화와 무관하다. 실제로 쓸 수 있는 건
수작업 큐레이션 640개뿐이다. 번역 정확도가 검증되지 않은 문장을 새로 만들지
않기 위해, 레슨 콘텐츠는 이 640개만 재활용한다.

## 데이터 성격 (중요)

큐레이션 640개는 **낱개 단어(608개) + 개별 표현(32개 sentence)** 구조이며, 실제로
주고받는 대화(dialogue) 스크립트가 아니다. 따라서 레슨의 핵심은 "대화문 통째
암기"가 아니라 **주제별 표현을 한 줄씩 듣고 따라하는 연습**이다.

다만 사용자 요청에 따라, 각 레슨 끝에 **간단 대화 예시**를 추가로 제공한다. 이
대화 예시는 **기존 640개 표현을 순서대로 배치(손님/직원 역할 부여)해 조립**한
것이며, 새 크메르어 문장은 만들지 않는다(번역은 100% 검증된 데이터만 사용).
대화체 표현이 충분한 레슨(인사·매장·시장·이동·긴급·기초표현)은 2~4줄 대화가
자연스럽게 나오고, 명사 위주 레슨(숫자시간·스몰토크·생활사물)은 대화 예시가
짧거나(간단 Q&A) 생략될 수 있다.

## 목표

- 탭 구조를 사전 / 학습 / 단어장 3개로 단순화 (회화 탭 제거, 학습 레슨에 통합)
- "학습" 탭을 SRS/퀴즈 방식에서 9개 상황별 레슨 코스로 교체
- 레슨은 자유롭게 넘나들 수 있고(잠금 없음), 완료 여부만 체크 표시
- 레슨 내 연습은 표현 한 줄 단위로 진행: 한국어 → 탭하여 크메르어/발음/존칭 공개 →
  음성 재생(mp3 우선, TTS 폴백) → 다음 줄. 레슨 끝에 전체 표현 복습 + 간단 대화
  예시 화면 제공
- 20,000단어 전체 사전 검색 기능은 그대로 유지 (하드 제약: 데이터 무손실)

## 비목표 (Non-goals)

- 새 크메르어 문장 생성/번역 (번역 검증 리스크로 인해 배제 — 대화 예시도 기존
  검증된 표현을 배치만 함)
- SRS 간격반복, 점수 매기는 퀴즈, streak 등 게이미피케이션 유지
- 레슨 순서 잠금

## 데이터 모델 변경

`scripts/build_dictionary.mjs`:
- `classifyLearningCategory(entry)`: 기존 7개 카테고리 매핑을 9개 레슨 ID로 교체.
  로직은 동일하게 유지 — `entry.source !== "수작업 기본 단어"`면 `null` 반환(사전
  전용, 학습 제외). 큐레이션 640개만 `entry.c` → 레슨 ID 직접 매핑.
- `payload.meta.learningCategories` → `payload.meta.lessons` (9개 배열, 각 항목에
  `id`, `title` 포함)로 이름 변경.
- `formality`(격식 3단계) 필드는 그대로 유지 — 레슨 내 대화 줄에 존칭 배지로 계속
  사용.

## UI/기능 변경

### 탭 구조
- 하단 네비게이션 4개 → 3개: 사전 / 학습 / 단어장
- 회화 탭(`renderPhraseView`, `phraseCard`) 제거, 관련 코드 삭제

### 학습 탭 (신규)
1. **레슨 목록 화면**: 9개 레슨 카드, 각 카드에 제목·표현 수·완료 진행률
   (예: "12/48 완료") 표시. 탭하면 레슨 상세로 이동.
2. **레슨 진행 화면**:
   - 표현이 순서대로 나열, 한 줄씩: 한국어 표시 → 탭하면 크메르어+
     한글발음+격식 배지 공개 + 음성 자동 재생(mp3 우선, 없으면 Web Speech API
     폴백) → "다음 줄" 버튼으로 진행
   - 각 줄은 "외웠어요" 토글 가능 (완료 진행률에 반영, 점수/정오답 없음)
   - 레슨 끝에 도달하면 (a) 전체 표현을 한 번에 보여주는 복습 화면(모든 줄
     펼쳐진 상태 + 각 줄 음성 재생 버튼)과 (b) **간단 대화 예시** 섹션을 표시.
     대화 예시는 손님/직원(A/B) 역할 라벨이 붙은 2~4줄 교환이며, 각 줄에 음성
     재생 버튼 제공. 대화 예시가 없는 레슨은 이 섹션을 숨김.
   - 잠금 없음 — 레슨 목록에서 언제든 어느 레슨이나 진입 가능
3. **localStorage**: 레슨별 완료한 줄 ID 집합만 저장 (기존 SRS box/streak 구조
   제거, 새 키 예: `khdict_lessons_v3`)

### 간단 대화 예시 데이터 (신규)
- `scripts/build_dictionary.mjs`에 `LESSON_DIALOGUES` 상수 정의: 레슨 ID별로
  `[{ role: "손님"|"직원", ko: "..." }, ...]` 형태의 대화 배열.
- 빌드 시 각 줄의 `ko`를 큐레이션 엔트리에서 조회해 `km`/`p`/`audio`를 붙인다.
  조회 실패(오타 등) 시 빌드가 에러로 중단 → 테스트가 잡는다. 이렇게 하면
  대화의 크메르어 번역은 항상 검증된 640개에서만 나온다(새 번역 생성 없음).
- 최초 대화 예시는 대화체 표현이 충분한 레슨 위주로 작성:
  인사(1), 매장/주문(4), 시장/흥정(5), 이동/택시(6), 긴급(7), 기초표현(3).
  명사 위주 레슨(2·8·9)은 초기엔 생략하거나 짧은 Q&A 1개만.

### 사전 탭
- 변경 없음. 카드/상세화면의 lessonCategory 배지는 9개 레슨 이름으로 자동 반영.

### 단어장 탭
- 변경 없음 (즐겨찾기 + 폴더 기능 유지)
